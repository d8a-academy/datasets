# Course datasets

Synthetic datasets used by the d8a.academy curriculum. They are generated by the
scripts in this folder and are meant to be hosted in a **public** GitHub repo named
`d8a-academy/datasets`, so the lessons' download links resolve.

> The lessons reference `https://raw.githubusercontent.com/d8a-academy/datasets/main/...`.
> If you host under a different account (e.g. `MariusMLD/datasets`), update the links in
> the lessons accordingly (search the content for `d8a-academy/datasets`).

## Generate

```bash
# one-off environment
python3 -m venv .venv && source .venv/bin/activate
pip install numpy pandas pyarrow

# Data Science datasets (the lumen_* files)
python3 generate_lumen_datasets.py --out out

# Data Analyst datasets (e-commerce CSVs + parquet)
python3 generate_da_datasets.py --out out/da
```

Everything is seeded (`SEED = 42`), so regenerating produces identical files.

## What to host where

Upload the generated files to the `d8a-academy/datasets` repo with this layout:

```
d8a-academy/datasets   (public)
├── lumen_users.csv              # DS 01 – descriptive stats        (~50k)
├── lumen_ab_test.csv            # DS 01 – hypothesis tests         (~20k)
├── lumen_subscriptions.csv      # DS 02 – churn classification     (~30k)
├── lumen_watch_target.csv       # DS 02 – regression               (~30k)
├── lumen_payments.csv           # DS 03 – fraud / feature eng.     (~80k)
├── lumen_streaming_daily.csv    # DS 04 – time series (4 regions)  (~6k)
├── lumen_reviews.csv            # DS 05 – NLP sentiment            (~30k)
└── da/
    ├── customers.csv            # DA 02/03/06                      (1.2k)
    ├── orders.csv               # DA 02/03/06 (intentionally messy)(~5k)
    ├── products.csv             # DA 02/03/06                      (200)
    ├── daily_revenue.parquet    # DA 07 – AI anomaly detection
    └── events/
        └── events_00..05.parquet  # DA 03 – DuckDB parquet glob demo
```

### Capstone release

The DS capstone (`09-capstone/02-problem-data.mdx`) downloads from a GitHub **release**
tagged `v1`, not from `main`. After pushing the files, cut a release:

```bash
gh release create v1 lumen_subscriptions.csv --repo d8a-academy/datasets \
  --title "v1" --notes "Lumen synthetic datasets v1"
```

## Schemas & signal

Each file matches the schema documented in its lesson, and carries realistic signal so
the analyses actually produce findings:

| File | Built-in signal |
| --- | --- |
| `lumen_ab_test` | ~+6% watch-time lift in `treatment`, balanced `signup_age_days`, ~50/50 split |
| `lumen_subscriptions` | churn (~11%) driven by low watch, short tenure, support tickets, auto-renew off |
| `lumen_watch_target` | next-month watch strongly autocorrelated with last month (r≈0.88), 0-spike |
| `lumen_payments` | fraud (~0.6%) driven by geo mismatches, night/fast charges, temp-mail |
| `lumen_streaming_daily` | trend + weekly/yearly seasonality + holiday/launch spikes, 4 regions |
| `lumen_reviews` | star distribution skewed to 4–5, 6 languages, sentiment-aligned text |
| `da/orders` | nulls, duplicates, mixed `status` casing, outliers (for the cleaning lesson) |

> Note: the North America region is labelled **`NAM`**, not `NA`, because `pd.read_csv`
> treats the literal `NA` as a missing value by default.
